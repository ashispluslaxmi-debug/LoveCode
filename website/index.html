📦 Installation

Get started in under a minute.

1

Install globally via npm

npm install -g lovecode-ai
2

Initialize in your project

lovecode init
3

Start chatting with AI

lovecode chat

📱 Termux (Android)

1

Install Node.js and lovecode

pkg install nodejs && npm install -g lovecode-ai
2

Initialize and start

lovecode init && lovecode tui
📱
LoveCode auto-detects Termux and enables low RAM mode (128MB), touch-friendly numbered input, and reduced cache TTL (120s).

🎨 Terminal UI

Rich split-pane TUI built with Ink + React. Full vim keybindings, themes, and streaming AI responses.

📐 Split Panes

Four panes: repo context, chat box, command runner, and input bar. Tab to cycle focus.

⌨️ Vim Mode

Esc for normal mode, i for insert, j/k scroll, g/G top/bottom.

🎭 5 Themes

Default, Dark, Light, Ocean (double borders), Solarized. Switch with /theme <name>.

💬 Markdown Rendering

AI responses render markdown with syntax-highlighted code blocks, tables, lists, and links.

⏎ Command History

Up/down arrows browse last 50 sent messages. Tab completion for slash commands.

💾 Session Persistence

Chat sessions auto-save. Export with /export, list with /sessions.

🔌 MCP Server

LoveCode exposes all 63 tools via the Model Context Protocol (MCP). Connect any MCP-compatible client to use LoveCode as a coding agent backend.

⚡ Quick Start

1

Install LoveCode

npm install -g lovecode-ai
2

Add to your MCP client config (Claude Desktop, VS Code, etc.)

{
  "mcpServers": {
    "lovecode": {
      "command": "lovecode",
      "args": ["mcp"]
    }
  }
}

⚙️ Configuration

LoveCode stores config in .lovecode/config.yaml. Managed via lovecode config or lovecode init.

# .lovecode/config.yaml model: codellama theme: neon approval_mode: smart provider: ollama model_params: temperature: 0.2 max_tokens: 4096 performance: lazy_load: true cache_ttl: 300000 max_memory_mb: 200 termux: low_ram: false touch_optimized: false telemetry: enabled: false security: profile: standard scan_secrets: true

🛠️ Development

LoveCode is built with TypeScript, Ink (React for terminal), and modern ESM.

📁 Project Structure

  • bin/ — CLI entry point
  • src/commands/ — Command implementations
  • src/tui/ — React Ink TUI components
  • src/ai/ — AI provider integrations
  • src/core/ — Agent and tool system
  • src/config/ — Configuration
  • src/security/ — Security sandbox
  • src/plugins/ — Plugin system
  • src/browser/ — Playwright automation
  • src/mcp/ — MCP protocol server
